## 核心模块详解

### 1. GenerationPipeline (核心流水线)

**职责**: 串联所有子系统，实现全自动小说生成流程

**工作流程**:
```
初始化 → 知识采集 → 世界观生成 → 大纲生成 → [章节生成循环] → 完成
                            ↓
                    ┌───────────────┐
                    │   主循环      │
                    │  1. 生成章节  │
                    │  2. 质量评估  │
                    │  3. 低分重写  │
                    │  4. 知识更新  │
                    │  5. 记忆维护  │
                    │  6. 大纲更新  │
                    └───────────────┘
```

**关键配置**:
- `outline_update_interval`: 大纲更新频率（每N章）
- `quality_threshold`: 质量阈值（低于此分数触发重写）
- `rewrite_threshold`: 重写阈值

### 2. 记忆系统 (MemoryManager)

**四层记忆架构**:
```
┌─────────────────────────────────────────────────────────────┐
│  永久记忆 (Permanent)  ~2K tokens  始终注入                │
│  ├─ 世界观核心设定                                         │
│  ├─ 主角核心档案                                           │
│  └─ 主线悬念 (S级)                                         │
├─────────────────────────────────────────────────────────────┤
│  长期记忆 (Long-term)  ~3K tokens  定期压缩                │
│  ├─ 卷摘要 (每10章压缩为200字)                             │
│  ├─ 角色弧线归档                                           │
│  └─ 已解决悬念归档                                         │
├─────────────────────────────────────────────────────────────┤
│  短期记忆 (Short-term) ~4K tokens  语义检索                │
│  ├─ 向量检索: 本章相关历史片段 (top 3-5)                   │
│  ├─ 角色关联检索                                           │
│  └─ 物品关联检索                                           │
├─────────────────────────────────────────────────────────────┤
│  工作记忆 (Working)   ~3K tokens  每章注入                  │
│  ├─ 上章结尾 500 字                                        │
│  ├─ 最近 5 章摘要链                                        │
│  ├─ 当前活跃悬念                                           │
│  └─ 本章大纲 + 关键事件                                    │
└─────────────────────────────────────────────────────────────┘
```

**Token预算管理**:
- `BUDGET_WORKING = 4500` (~3000 tokens)
- `BUDGET_SHORT_TERM = 6000` (~4000 tokens)
- `BUDGET_LONG_TERM = 4500` (~3000 tokens)
- `BUDGET_PERMANENT = 3000` (~2000 tokens)

### 3. 向量存储 (VectorStore)

**功能**:
- 文本向量化 (sentence-transformers)
- 向量索引 (JSON持久化 + numpy计算)
- 语义相似度检索 (余弦相似度)

**性能优化**:
- 模块级单例: 模型只加载一次
- 后台预加载: 与MySQL初始化并行
- 批量编码: 大文档自动分批

### 4. 资产管理系统

**角色管理 (CharacterManager)**:
- 档案创建与更新
- 状态追踪 (活跃/死亡/失踪/退出)
- 人际关系图谱
- 高光时刻记录

**物品管理 (ItemManager)**:
- 物品追踪 (持有/损毁/赠送/遗失)
- 品阶分类
- 剧情意义评估
- 防膨胀机制

### 5. 悬念管理系统

**悬念分级**:
- **S级**: 主线悬念（贯穿全书）
- **A级**: 卷级悬念（当前地图核心谜团）
- **B级**: 章级悬念（短期可解决）

**状态管理**:
- `ACTIVE`: 活跃中
- `RESOLVED`: 已解决
- `ABANDONED`: 已放弃

### 6. Skill系统

**匹配策略**:
1. 使用指定的 skill_id
2. 从 Registry 自动匹配预置 Skill
3. 用 LLM 自动生成 Skill
4. 使用 fallback _base Skill

## 错误处理机制

### 异常层次结构

```
NovelAgentError (基础异常)
├── DatabaseError (数据库异常)
├── LLMError (LLM调用异常)
├── ValidationError (验证异常)
├── ConfigurationError (配置异常)
├── PipelineError (流水线异常)
│   ├── ChapterGenerationError
│   ├── QualityAssessmentError
│   └── OutlineError
├── KnowledgeError (知识采集异常)
├── VectorStoreError (向量存储异常)
├── SkillError (Skill系统异常)
├── MemoryError (记忆系统异常)
├── SuspenseError (悬念管理异常)
└── AssetError (资产管理异常)
```

### 统一错误处理函数

```python
handle_error(
    error=exception,
    context="错误上下文描述",
    raise_error=False,  # 是否重新抛出
    default_value=None, # 出错时返回的默认值
)
```

## 配置管理

### 环境变量配置 (.env)

```bash
# MySQL 配置
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=your-password
MYSQL_DATABASE=agnes_novel

# LLM 配置
AGNES_API_KEY=sk-xxxxx
AGNES_BASE_URL=https://apihub.agnes-ai.com/v1
AGNES_MODEL=agnes-2.0-flash

# 生成参数
DEFAULT_CHAPTERS=100
WORDS_PER_CHAPTER=8000
WORDS_PER_SEGMENT=3000

# 向量数据库
VECTOR_DB_PATH=./vector_db
CHUNK_SIZE=2000
CHUNK_OVERLAP=200
```

### Pydantic配置模型

- `LLMConfig`: LLM模型配置
- `MySQLConfig`: 数据库配置
- `GenerationConfig`: 生成参数配置
- `VectorDBConfig`: 向量数据库配置
- `SchedulerConfig`: 任务调度配置
- `SkillConfig`: Skill系统配置

## 性能优化策略

1. **并行初始化**: embedding模型预加载与MySQL初始化并行
2. **批量编码**: 大文档自动分批编码，避免内存溢出
3. **向量缓存**: numpy矩阵缓存，避免重复计算
4. **延迟加载**: 子系统实例按需创建
5. **连接池**: SQLAlchemy连接池管理
